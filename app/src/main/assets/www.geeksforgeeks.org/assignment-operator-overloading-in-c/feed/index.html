]]> By: Kartik http://www.geeksforgeeks.org/assignment-operator-overloading-in-c/#comment-11404 Thu, 04 Oct 2012 12:53:42 +0000 http://www.cdn.geeksforgeeks.org/?p=24761#comment-11404 C++ doesn’t allow default assignment operator to be used when there is a reference in your class.

For example, the following program produces error “error: non-static reference member ‘int& Test::ref’, can’t use default assignment operator”

 
#include<iostream>
using namespace std;

class Test
{
    int x;
    int &ref;
public:
    Test (int i):x(i),ref(x) {}
    void print() { cout << ref; }
    void setX(int i) { x = i; }
    //Test &operator = (const Test &t) {x = t.x;}
};

int main()
{
    Test t1(10);
    Test t2(20);
    t2 = t1;
    t1.setX(40);
    t2.print();
    return 0;
}

If you uncomment the operator definition in the above program, the program works fine and prints “10”.
So compiler forces to write an assignment operator when you have a non-static reference in your class.
