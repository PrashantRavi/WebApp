]]> By: Venki http://www.geeksforgeeks.org/g-fact-51/#comment-2439 Thu, 11 Nov 2010 05:07:02 +0000 http://geeksforgeeks.org/?p=9040#comment-2439 @Sandeep and @Raja, switch statement is used in lieu of large if-else ladder when an identifier is compared against a set of integral constants. (I guess float constants is not allowed in case labels, only integrals are allowed)

For example, a system state can be enumerated in a set of enumerated constants. The programmer would need to determine the current system state and process accordingly. In such case, it is usual to use if-else ladder, but results in poor performance.

switch is another alternative that follows a table loop-up approach in determining correct if-else clause.

The compiler makes read-only switch case ‘constants table’ in non-decreasing order and invokes binary search on the table to determine exact case statement. It improves performance a lot. Due to this fact, most of the compilers place the switch tables in read-only memory. An attempt to modify such case integral value can result in runtime error. It is suggested only integral constants (integers, enumerated constants, macros, etc.) to be used as case labels.

It is not recommended to use const in case labels. const variables are evaluated at compile time. Though the following code is not practical, it shows how to break usage of const in case labels (or even may result in exception on some platforms),

 
int main()
{
    const int i = 10;
    int *p = (int *)&i;
    int c = 10;
    
    ++*(int *)&i;

    switch(c)
    {
    case i:  // Works fine
        printf("Value of c = %d i = %d *p = %d", c, i, *p);
        break;

    default:
        printf("From default case %d", *p);
    }

    return 0;
}

Guess what will be the output and why?
