]]> By: Mragank Yadav http://www.geeksforgeeks.org/union-find/#comment-37563 Tue, 03 Jun 2014 20:58:00 +0000 http://www.cdn.geeksforgeeks.org/?p=26350#comment-37563 For all those who are thinking that calling find() function again in Union() is redundant,

It is not and is absolutely mandatory.
Here understand it with the following example:
Consider this your graph:
0—-1–2
……. | /
……..3

now our parent would be:
0 1 2 3
-1 -1 -1 -1

On passing Edge (0-1), parent becomes
0 1 2 3
1 -1 -1 -1

Edge (1-2) gives

0 1 2 3
1 2 -1 -1

Edge (2-3) gives

0 1 2 3
1 2 3 -1

Now, When Edge (1-3) comes, if there is no find() in Union(), then our resulting parent would be

0 1 2 3 which would be wrong.
1 3 3 -1

But cause of find(), we know that Vertex 1 belongs to the subset {0,1,2}

and so parent would become

0 1 2 3 which would correctly detect the cycle 1->3->2.

1 2 3 -1

I hope i am correct and this avoids confusion.
