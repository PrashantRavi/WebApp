]]> By: Rakesh Gupta http://www.geeksforgeeks.org/g-fact-75/#comment-18614 Fri, 24 May 2013 21:06:53 +0000 http://geeksforgeeks.org/?p=9788#comment-18614 It is not mandatory to collect the return value of the functions in a function call expression. Consider the following code,

 
int fun();
char fun();
int main()
{
    fun();
}

Here it’s not possible to predict which function would be called. Hence compiler will generate compile time error.

Also during name mangling, only function parameters, their types and sequence are used to calculate mangled name, not the return type. Hence two functions, who differ in return type only, will have same mangled name which is invalid. Two functions cannot have same mangled name(internal representation).
