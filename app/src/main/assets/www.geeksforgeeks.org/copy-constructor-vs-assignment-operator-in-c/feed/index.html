]]> By: Bidyapati http://www.geeksforgeeks.org/copy-constructor-vs-assignment-operator-in-c/#comment-4725 Wed, 27 Jul 2011 02:27:32 +0000 http://geeksforgeeks.org/?p=8229#comment-4725 I think you are mixing up assignment operator and copy constructor. They are used in different scenarios.
Copy constructor is called:
1)When object is initializing from another object:
A a(b);
A a=b;
2)During function call as a local object creation in stack:
int fun(A obj)
3)During return from a function
A fun()

Assignment operator is called when both the objects are already created and we are assigning one to other.
a = b

Now about member variable pointers, if they are initiallized with a heap memory and later a new object created using copy constructor or assigned to another object(both the case), now both the pointers(new object and old) pointing to same heap memory. When object is deleted and the pointer is freed in destructor, it creates a dangling pointer(member variable of other object). This can happen in assignment or copy constructor.
So in these scenario, we need to explicitly define these operators, we should not rely upon compiler.
