]]> By: ptr http://www.geeksforgeeks.org/calloc-versus-malloc/#comment-4833 Sun, 07 Aug 2011 07:25:52 +0000 http://geeksforgeeks.org/?p=5163#comment-4833 For malloc() linux OS (and many other variants) use optimistic memory allocation. So pointer returned by malloc is not backed by physical memory until user uses that memory. So even if malloc returns NULL there is a chance that memory may not be available in out of memory conditions (OOM).

In some linux systems Calloc() gets a chunk of memory which is already initialized to 0. When CPU is idle OS may initialize unallocated memory to zero and same memory block is returned by Calloc call.

Malloc should be used in scenarios where memory allocated is going to be used(overwritten) immediately by the application.
There is no need to initialize that memory to zero.

Calloc should be used in cases where part of memory allocated will be used at later point of time(atleast not immediately) and its better to initialize unused memory to zero.
