So I look into the bits myself and I found a pattern that explained why the algorithm works. Take a look at the value of each bit when it’s 1:
1 = 3*0 + 1
2 = 3*1 -1
4 = 3*1 + 1
8 = 3*3 -1
16 = 3*5 +1
32 = 3 * 11 -1

You can see they alternate between (3*x +1) and (3*x -1). So if number of odd set bits and number of even sets bits are the same, then they cancel out each other, hence it’ll be divisible by 3. Same conclusion if the difference between the two numbers is multiple of 3. Since we are looking at 32 bits integers, then the biggest difference in the two numbers would be 16 (e.g. all odd bits are 1 and all even bits are 0), so we just need to check if the result is 15, 12, 9, 6, 3, or 0.
