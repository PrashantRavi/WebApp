So in essence the values of 'l' and 'r' are constantly being divided such that the 'r' index eventually aligns with the first index of key in the array. But consider the case when there are no multiple occurrences of key in the array but the key is present only at the boundaries of the array. In that case, even though the invariant remains true all the time, but the while loop condition will prevent 'r' from moving to the leftmost index of key, the same issue will be seen when we are trying to find the rightmost occurrence of a key which occurs only once and is at the right end of the array.
