]]> By: Mittal http://www.geeksforgeeks.org/write-a-function-to-get-the-intersection-point-of-two-linked-lists/#comment-38958 Tue, 17 Jun 2014 15:18:00 +0000 http://geeksforgeeks.org/?p=2405#comment-38958 Well, I think that is not possible.

Consider the list represented as M+C and N+C where C is the common part.

d = | (M+C) – (N+C)| = |M-N|

So the common part is always cancelled out. What d gives is the number of nodes by which one list is longer, if at all it is (as d=0 is also possible)

Moving d nodes down the longer list, say M > N here, (M+C) – (M-N), you are left with N+C nodes in longer list which is same as the shorter one. Hence they can then be traversed in parallel
