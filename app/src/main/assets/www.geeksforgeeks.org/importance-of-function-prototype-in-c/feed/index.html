]]> By: saurav http://www.geeksforgeeks.org/importance-of-function-prototype-in-c/#comment-9556 Thu, 12 Jul 2012 04:38:58 +0000 http://www.cdn.geeksforgeeks.org/?p=21814#comment-9556 Hi Ravi,

gcc doesn’t bother much about the function prototype. It may give warning if it doesn’t find the prototype of function (but this is an error for g++). GCC hope it will find the function definition at link time and if it doesn’t find that causes a linker error. The interesting part is you can call a function (whose prototype is not mentioned) with certain number of parameters, which is actually defined with some different no. of parameters but still compiler will compile successfully and obviously you’ll not get the expected result. The point is linker only looks for the definition against a name and link it.
