]]> By: Molly Poppins http://www.geeksforgeeks.org/understanding-extern-keyword-in-c/#comment-38975 Tue, 17 Jun 2014 19:18:00 +0000 http://geeksforgeeks.org/?p=840#comment-38975 Not sure if this came up already, didn’t read all the comments, but Example 4 will compile and run without “extern int var;”.
e.c:
#include
#include "somefile.h"

int main()
{
var = 10;
printf("%dn", var);
return 0;
}

somefile.h:

#ifndef SOMEFILE_H
#define SOMEFILE_H

int var = 10;

#endif

Because the #include directive puts all the code in somefile.h into e.c. Watch what happens when i just run the preprocessor:

$ cpp e.c e.i
$ cat e.i
# 1 "e.c"
# 1 ""
# 1 "e.c"
# 1 "somefile.h" 1

int var = 10;
# 2 "e.c" 2

int main()
{
var = 10;
return 0;
}

I got rid of the C library stuff to make the example smaller. But you see how “extern int var;” isn’t necessary because the var definition gets added to the .c file by the preprocessor anyway. A better example might be if you had a seperate .c file with the var definition:
$ cat e.c

extern int var;

int main()
{
var = 10;
return 0;
}

$ cat e2.c
int var = 0;

$ gcc -o e e.c e2.c

It compiles fine. Hope this helps.
