]]> By: Vivek http://www.geeksforgeeks.org/how-can-i-return-multiple-values-from-a-function/#comment-4553 Thu, 07 Jul 2011 13:21:17 +0000 http://geeksforgeeks.org/?p=955#comment-4553 Although vaguely I do agree with the presented facts, there is not a certain binding that one should use pointer types ONLY to return multiple values.

Heres an example returning multiple values encapsulated inside a struct:

 
#include <iostream>
using namespace std;

struct tagstEmployee
{
	int 	iEmpId;
	string 	strEmpName;
};

tagstEmployee my_func ( void )
{
	tagstEmployee empdetails;
	empdetails.iEmpId = 007;
	empdetails.strEmpName = "James Bond";
	return empdetails;
}

int main(void)
{
	tagstEmployee local = my_func();

	cout << "[" << local.iEmpId << "] = " <<  local.strEmpName;
}

– Here we are returning a composite type instead of just one. Thus in a sense what the original poster meant was we are able to return multiple values.
– Right now we are having just two primitive data to be returned. In real world scenario this would contain large data and we might be smart enough to return pointers, and hence create the data on a heap.
– Also instead of just returning "multiple values" via struct or some other mechanisms, we can pass the object pointer by address as a function argument itself! Something on the lines of void my_func ( struct tagstEmployee *& emp )

Different ways, same goals 🙂
