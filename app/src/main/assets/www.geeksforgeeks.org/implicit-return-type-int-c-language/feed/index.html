]]> By: Meet http://www.geeksforgeeks.org/implicit-return-type-int-c-language/#comment-31856 Sun, 26 Jan 2014 09:44:00 +0000 http://www.cdn.geeksforgeeks.org/?p=125702#comment-31856 Hey, kartik you are right. But in older C++ compilers if we don’t supply return type of a method C++ compilers assumes default return type as int. But implicit int return type has been banned by both C99 and C++. As an experiment try to compile this program in older C++ compiler.
#include
class Test
{
int s;
public:
get()
{
cout<>s;
}
show()
{
cout<<"s = "<<s<<endl;
}
};
int main()
{
Test t;
t.get();
t.show();
return 0;
}// object goes out of scope
Try this program in older C++ compiler, this program compiles and runs without error. Don't try to compile this program in any modern C++ compiler which is ANSI/ISO standard.
