]]> By: Hongliang http://www.geeksforgeeks.org/print-0-and-1-with-50-probability/#comment-11058 Sat, 22 Sep 2012 10:18:55 +0000 http://www.cdn.geeksforgeeks.org/?p=24033#comment-11058 em, this one does not work, actually. For example, we made two tosses, they are (0, 0), and (0, 0), this one simply fails. One can start over, but this algorithm can not guarantee the result.

Some good solutions can be found online. The famous one is from John von Neumann http://en.wikipedia.org/wiki/Fair_coin#Fair_results_from_a_biased_coin

”
Toss the coin twice.
If the results match, start over, forgetting both results.
If the results differ, use the first result, forgetting the second.
”
The discussion for this algorithm is here:
http://stackoverflow.com/questions/5429045/c-puzzle-make-a-fair-coin-from-a-biased-coin

It is not so efficient. There are some further discussions, for example, this tree based algorithm gives one linear (almost) solution:

http://web.eecs.umich.edu/~qstout/abs/AnnProb84.html
