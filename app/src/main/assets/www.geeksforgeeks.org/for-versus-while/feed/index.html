]]> By: ptr http://www.geeksforgeeks.org/for-versus-while/#comment-4831 Sun, 07 Aug 2011 05:47:27 +0000 http://geeksforgeeks.org/?p=3850#comment-4831 @sachin: compiler is free to do its own optimizations for such code. In my system while(1) and for(;;) results in same assembly code.

C Code for while(1):

 
#include <stdio.h>
int main()
{
   while(1){}
}

Assembly:

.globl main
	.type	main, @function
main:
	pushl	%ebp
	movl	%esp, %ebp
.L2:
	jmp	.L2

And even this code using for(;;) loop produced same assembly code:

 
#include <stdio.h>
int main()
{
   for(;;)
   {}
}

Assembly:

.globl main
	.type	main, @function
main:
	pushl	%ebp
	movl	%esp, %ebp
.L2:
	jmp	.L2

So my compiler knew what i’m doing!

Note:
you can check assembly code using this command
gcc -S example.c
which will give an assembly file example.s in same directory.

]]> By: Venki http://www.geeksforgeeks.org/for-versus-while/#comment-1916 Sun, 08 Aug 2010 17:47:52 +0000 http://geeksforgeeks.org/?p=3850#comment-1916 For and while loops essentially do same operation. The for loop provides more readability and maintainable code. While is used for simple body. Traditionally most of thread functions are designed over while(1) expression.

The for loop provides little benefit in terms of “locality of reference”, means the initialization expression, conditional expression and update expression all can be located near vicinity and more probable to access from cache.
