]]> By: Arjun Suresh http://www.geeksforgeeks.org/g-fact-59/#comment-33260 Thu, 06 Mar 2014 19:55:00 +0000 http://geeksforgeeks.org/?p=9254#comment-33260 Its following both precedence and associativity rules of C. But still output is undefined because of the following reason:

Suppose we write

b = ++a + a;

the precedence rule says that ++ is to be done before +. So, the increment value of a will always be used for the plus. But, now the right hand side should take the incremented value or the unincremented value? ++a had a side effect of modifying the variable a, but this modification may not be done immediately. It’ll be done only before the next sequence point (for simplicity take sequence point as next “;”). Because of this reason this code can produce different outputs.. And this is clearly mentioned in C standards..

Why can’t any one write?
++a;
b = 2*a;
instead of b = ++a +a;
