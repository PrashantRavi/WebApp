]]> By: YouKnowWho http://www.geeksforgeeks.org/analysis-algorithm-set-4-master-method-solving-recurrences/#comment-35990 Tue, 13 May 2014 16:30:00 +0000 http://www.cdn.geeksforgeeks.org/?p=125673#comment-35990 This cannot be done using Master method. We have exponential number of subproblems being generated at each step.
Let us say T(1) = 1
Then,
T(n) = 2^n*(T(n/2)) + n^n
= 2^n*(2^(n/2)*T(n/4) + (n/2)^(n/2)) + n^n
<= 2^(n+ n/2 + n/4 + …) + n^n + (n/2)^n + (n/4)^n + ….
and so on..
This would clearly be O(n^n) since the first term would become 2^(n*(1-(1/2)^log(n))/(1-0.5))) = 2^(2*(n-1))
The second term would boil down to n^n(1 + (1/2)^n +(1/4)^n + …)
This would be equal to (n^n)*(1 – ((0.5)^nlogn)/(1 – 0.5^n) = n^n(1 – (nlogn)^-1)/(1-0.5^n)
= ((2n)^n)*(nlogn – 1)/(2^n – 1) <= 2n^n after some n0 thus, this would be in O(2n^n).
You could achieve much stricter upper bounds with more analysis.
