]]> By: Shriram http://www.geeksforgeeks.org/what-is-the-purpose-of-a-function-prototype/#comment-2152 Tue, 07 Sep 2010 00:30:09 +0000 http://geeksforgeeks.org/?p=1061#comment-2152 I feel that you can order the definition of your function any how if you declare the function prototype since the compiler knows before hand looking at the prototype that it is going to encounter the function at some point of time. For eg:

 
#include<stdio.h>

void foo()
{
   printf("foo called");
}

int main()
{
   foo();
   getchar();
   return 0;
}

would work fine since when foo() is called the compiler has already seen the foo() defined above.
However, the following will not work,

 
#include<stdio.h>

int main()
{
   foo();
   getchar();
   return 0;
}
void foo() /*Not OK since we do not have a prototype defined */
{
   printf("foo called");
}

In order for this to work, you insert a function prototype for foo() so that you can define it anywhere in the file and not necessarily before the point it is called. Hence, this would work fine:

 
#include<stdio.h>

void foo(void); /* function prototype */
int main()
{
   foo();
   getchar();
   return 0;
}
void foo() /* function is defined after the point it is called. It is OK since we have a function prototype. */
{
   printf("foo called");
}
