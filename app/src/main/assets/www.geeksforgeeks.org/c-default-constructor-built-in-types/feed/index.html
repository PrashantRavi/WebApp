C++ allows even built-in type (primitive types) to have default constructors. The function style cast int() is analogous to casting 0 to required type. The program prints 0 on console.

Wrong! C++ doesn’t allow built-in types to have default constructors.

There is something known as Value Initialization in C++(introduced in C++03) on account of which `int()` yields 0.

Section 8.5/5
To value-initialize an object of type Tmeans:
â€” if T is a class type (clause 9) with a user-declared constructor (12.1), then the default constructor for Tis
called (and the initialization is ill-formed if Thas no accessible default constructor);
â€” if T is a non-union class type without a user-declared constructor, then every non-static data member
and base-class component of Tis value-initialized;
â€” if T is an array type, then each element is value-initialized;
â€” otherwise, the object is zero-initialized

`int ()` yields zero because of the last statement “otherwise, the object is zero-initialized”.

Also check out the definition of `zero initialization` in C++03.
