]]> By: kartik http://www.geeksforgeeks.org/are-array-members-deeply-copied/#comment-4887 Fri, 12 Aug 2011 04:28:44 +0000 http://geeksforgeeks.org/?p=13899#comment-4887 @Agniswar: We don’t need copy constructor for arrays, we need them when we have pointer(s) in out class. Following is an example of string class where we must write copy constructor. 

 
class string { 
 private: 
   char *s; 
   int size; 
 public: 
   string(char *);             // constructor 
   string(const string&); // copy constructor 
}; 

 
string::string(char *c)
{
 size = strlen(c);
 s = new char[size+1];
 strcpy(s,c);
}

string::string(const string& old_str)
{
 size = old_str.size;
 s = new char[size+1];
 strcpy(s,old_str.s);
}

For the above string class, we must also overload the assignment operator and write a destructor that deletes dynamically allocated memory.
