]]> By: David http://www.programcreek.com/2014/06/leetcode-count-complete-tree-nodes-java/#comment-7062 Fri, 19 Jun 2015 02:19:00 +0000 http://www.programcreek.com/?p=14487#comment-7062 In the worst case, you will have to keep making recursive calls to the bottom-most leaf nodes (e.g. last level only have one single node). So you end up calling countNodes() h times. Each time, you will have to do traversals along the left and right edges. At level h, you iterate zero times (no child). At level h – 1, you iterate once (one child). And so on. So that is 0 + 1 + 2 + … + h steps just to compute the left edges, which is h(1 + h)/2 = O(h^2).

The space complexity will just be the size of the call stack, which is O(h).
